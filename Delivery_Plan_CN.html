1项目概述

本项目为 Alinma Bank 设计、实施、集成、测试、部署并支持一个 GenAI 平台，整合三项协同能力：

  • M1 — AI 外呼电话营销
  • M2 — 数字渠道潜在客户的 AI 跟进
  • M3 — AI 优先的呼入联络中心（阶段 1 咨询、阶段 2 投诉）

整体交付周期 5 个月。所有 AI 工作负载与客户交互运行在 Alinma 控制的 KSA 境内本地基础设施。

2交付范围

2.1 我方核心交付（5 个月内）

  • 平台软件全栈：AgentStudio（低代码 Agent 构建 + LLM 推理引擎）、Voice SaaS、ASR / TTS（阿语 + 英语）、RAG 管道 + 知识库管理、变更治理能力、集成消费端代码、数据契约 + EDW 推送。
  • 3 个 UAT Bot（M1 / M2 / M3 阶段 1 各 1 个）：端到端、可投产，作为完整 Bot 的骨架版（见 2.2）。
  • 培训：4 类受众（见 §9.1）。
  • Hypercare：上线后超期支持。
变更治理能力：对机器人可配置内容（对话流、提示词、知识内容、模型选择等）的变更，通过 diff 展示 + maker-checker 双岗审批 + 版本控制 + 一键回滚 + 审计日志进行管理。具体治理方案（审批层级、覆盖范围）在实施阶段与贵行对齐后确定。

2.2 UAT Bot 模式

我方交付平台工具 + 3 个 UAT Bot（M1 / M2 / M3 阶段 1 各 1 个）。每个 Bot 在最小对接范围内打通端到端流程（ESB + 关键业务服务 + IVR/SIP + ASR/TTS/LLM + 知识库 + 转人工），用于端到端验收并作为生产 Bot 的初始版本。每个 UAT Bot 在 P2 与业务部门共同设计，覆盖业务场景、集成目标（T24/CRM/Comarch）、身份识别与验证（ID&V）三个维度。

ID&V（身份识别与验证）是电话场景的关键环节。受 RFP INT-27 约束，认证逻辑由贵行集成层服务提供、AI Agent 不自带认证。我方将在实施阶段，根据贵行现有 ID&V 能力与设想，对呼入 / 呼出场景的认证逻辑进行针对性调研与方案设计。

UAT Bot 覆盖范围（对应 RFP §12 UAT 要求）：RFP §12 规定的 UAT 场景，由 3 个 UAT Bot 的设计直接承载——

RFP §12 UAT 覆盖项对应承载

这一覆盖范围决定了 UAT Bot 的对话设计、集成、搭建、测试与验收工作量（见 §6 与人力投入附录）。

2.3 扩展模式（启动会后讨论确认）

  • 每模块可扩展至 2–3 个 UAT Bot（范围扩展需评估对时间表的影响）。
  • M1 / M2 全产品线、M3 阶段 2 的完整 Bot 由 Alinma 用我方平台自搭，我方提供平台、方法论、培训与 Hypercare 期支持。
  • 长期协作模式由后续单独商讨。

8关键假设

本计划以下列假设为前提；如不成立，启动变更控制流程重新评估。

范围边界

  • M1 / M2 全产品线、M3 阶段 2 的完整 Bot 由 Alinma 用我方平台自搭（见 2.3）。
  • 客户已自有外呼调度 + 全渠道营销系统（推定含 Comarch），我方仅集成、不替代。
  • 业务规则（资格 / 转接 / 投诉分类 / 强制人工）与知识内容由 Alinma 业务方提供。
  • Alinma 现有 IVR / 联络中心 / 拨号器可对接（不替换、不重构，仅 SIP 级接入）。
  • 多步业务流程编排由 Alinma 现有编排能力或集成层承担，AgentStudio 仅承担对话流编排。
  • EDW 端 ETL / 建模 / 报表 / 归档由 Alinma EDW 团队负责，我方提供数据契约与推送。
  • 硬件物料清单与采购由本地 SI 提供。

客户 readiness

  • Alinma 已具备 ESB + 粗粒度业务服务 + 共享日志 / 安全 / 错误处理服务。
  • Alinma 已有 IAM/SSO（OAuth2/OIDC）、KMS、SIEM、APM 等共享基础设施。
  • Alinma 提供真实通话样本（沙特方言含标注）用于语言能力评估与演示（非用于模型训练）。
  • Alinma 内部已批准本项目 budget 与 headcount。

时间与环境

  • 项目实际启动日 ≤ 2026-09-01（避开 2027 斋月）。
  • 开发环境采用境外团队远程接入境内开发环境的方案（待与贵行合规对齐）。
  • ASR / TTS / LLM 以现有产品级能力直接使用，本项目不含针对客户数据的模型训练或调优；语言能力以实际效果进行评估，验收标准由双方在测试时按全球基准共同确定（呼应 RFP §12）。
